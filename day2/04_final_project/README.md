# 실습 4 - Copilot을 활용한 나만의 프로젝트 2 + 최종 발표

## 목표
지금까지 배운 Custom Instructions, Agent Skills를 **내가 실제로 만든 프로젝트**와
**내 업무/관심 분야**에 적용해봅니다. 정해진 정답이 없는 자유 실습입니다.

## 실습 시간
약 90분 (자유 실습 60분 + 발표 준비/발표 30분 권장)

---

## 진행 방식
아래 3가지를 순서대로, 또는 시간 배분에 맞게 자유롭게 진행합니다.

### 1. 1차 프로젝트 결과물에 대한 나만의 Custom Instruction 만들기
- Day1에서 만든 나만의 TODO 앱(또는 Day2 실습 중 가장 애착이 가는 프로젝트)을 엽니다.
- 그 프로젝트를 위한 `.github/copilot-instructions.md`를 직접 작성합니다.
- 처음부터 손으로 다 쓰기 어렵다면, Copilot에게 초안 작성을 요청해도 됩니다.
  ```
  현재 프로젝트 구조와 코드를 분석해서
  .github/copilot-instructions.md 초안을 만들어줘.
  파일 역할 분리, API 규칙, 지켜야 할 컨벤션을 포함해줘.
  ```
- 생성된 초안을 그대로 쓰지 말고, 실습 1(`01_custom_instructions`)에서 배운 기준으로 직접 검토하고 다듬습니다.

### 2. 본인 업무에 도움이 될 나만의 Skill 만들기
- 지금 프로젝트에 국한되지 않고, **평소 본인 업무에서 반복되는 작업**을 떠올려봅니다.
  - 예: 코드 리뷰 체크리스트, API 문서화 형식, 테스트 코드 작성 가이드, 장애 대응 절차 정리 등
- 실습 2(`02_agent_skills`)에서 배운 구조로 `.github/skills/<my-skill>/SKILL.md`를 만듭니다.
  - `description`에는 "언제 이 Skill을 써야 하는지"를 최대한 구체적으로 적습니다.
  - 필요하면 `references/`(상세 문서)나 `scripts/`(실행 스크립트)도 함께 구성합니다.
- 만든 Skill이 실제로 발견→활성화되는지 관련 요청을 보내 확인합니다.

> **아이디어를 못 정하겠다면**: 각자 만든 프로젝트가 전부 다르기 때문에 "이런 Custom
> Instructions/Skill을 만드세요"라고 미리 정해줄 수 없습니다. 대신 Copilot Chat에게
> 이렇게 물어보세요. (오늘 배운 것과 별개로 실무에서도 그대로 쓸 수 있는 방법입니다)
> ```
> 이 프로젝트 코드를 보고, 앞으로 작업하면서 반복될 것 같은 요청 3가지를
> 추천해줘. 그리고 그중 프로젝트 전체에 항상 적용되면 좋을 규칙(Custom
> Instructions 후보)과, 특정 작업에서만 불러 쓰면 좋을 지침(Skill 후보)을
> 구분해서 제안해줘.
> ```
> 제안받은 것 중 실제로 본인 프로젝트에 맞는 것을 하나씩 골라서 다듬으면 됩니다.
> (프로젝트 성격에 따라 완전히 다를 수 있습니다 — 예: 웹 앱이면 "API 응답 형식
> 통일", 데이터 분석 코드면 "차트 스타일 통일", 자동화 스크립트면 "로그 포맷
> 통일" 등)

### 3. 자유 프로젝트 수행
- 앞서 만든 Custom Instructions/Skill을 활용해서, **본인의 업무나 관심 분야와 관련된 프로젝트**를 자유롭게 진행합니다.
- 지금까지 배운 것을 최대한 조합해서 사용해봅니다.
  - TCCOV 프롬프트로 세부 요청 → Custom Instructions로 프로젝트 규칙 고정 → Agent Skills로 반복 작업 자동화 → awesome-copilot에서 참고할 만한 예시가 있는지 확인
- 막히면 언제든 Copilot Chat에게 물어보고, 에러가 나면 에러 메시지를 그대로 붙여넣어 도움을 요청합니다.

## 제출 및 발표

### 전원: 양식 작성 및 제출
참가자 전원은 정리 양식(워드 파일)을 작성해서 제출합니다.
- 무엇을 만들었는가? (한 줄 소개)
- 어떤 Custom Instructions/Skill을 만들었고, 왜 그렇게 만들었는가?
- Copilot이 예상과 다르게 동작했던 순간이 있었다면, 어떻게 해결했는가?
- 이틀간 배운 것 중 실무에 가장 먼저 적용해보고 싶은 것은 무엇인가?

양식을 채우다 보면 자연스럽게 발표 내용이 정리됩니다.

### 발표자 선정 및 발표
- 참가자 투표로 1~2명을 발표자로 선정합니다.
- 발표 시간은 1인당 2~3분입니다.
- 순서 제안: (1) 무엇을 만들었는지 한 줄 소개 → (2) 화면으로 실제 동작 시연 → (3) 이번 프로젝트에서 만든 Custom Instructions/Skill 중 하나 간단히 보여주기 → (4) 배운 점 한마디
- 완성도보다 **Copilot을 어떻게 활용했는지**에 초점을 맞춰 발표합니다.

## 핵심 포인트
> 정답이 없는 실습입니다. 이틀간 배운 TCCOV, Custom Instructions, Agent Skills,
> awesome-copilot 활용을 실제 내 업무/관심 분야에 조합해서 써보는 것 자체가 목표입니다.
