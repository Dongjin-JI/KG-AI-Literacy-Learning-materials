# 새 API 엔드포인트 추가 절차

이 프로젝트에 새로운 API 엔드포인트를 추가할 때는 아래 절차를 따릅니다.
(빈칸을 직접 채워보세요.)

## 1. 파일 역할 분리
- (예: DB 쿼리 로직은 반드시 database.py 에만 작성한다)
- (예: main.py 에는 라우팅과 요청/응답 처리만 작성한다)

## 2. 응답 형식
- (예: 성공 시 {"message": "..."} 형태로 응답한다)
- (예: 기존 엔드포인트와 동일한 응답 스키마를 유지한다)

## 3. 반드시 지킬 것
- (예: SQL 쿼리는 파라미터 바인딩만 사용한다)
- (예: DB 커넥션은 사용 후 반드시 닫는다)

## 4. 상세 체크리스트
- (예: 입력값이 비어있거나 잘못된 형식일 때 422 응답을 반환하는가)
- (예: 존재하지 않는 id를 요청하면 404를 반환하는가)
- (예: 트랜잭션이 필요한 작업은 커밋 전에 검증을 마쳤는가)
- (예: static/index.html의 fetch 호출부도 함께 수정했는가)
