## 나의 개발 환경
- 주력 언어:         # 예: Java 17
- 프레임워크:        # 예: Spring Boot 3.x
- DB:               # 예: MySQL 8.0, Redis
- 빌드 도구:         # 예: Gradle
- IDE:              # 예: IntelliJ IDEA

## 현재 프로젝트
- 프로젝트 설명:     # 예: 사내 인사 관리 시스템 백엔드 개발
- 팀 규모:          # 예: 백엔드 3명, 프론트 2명
- 주요 도메인:      # 예: 결제, 회원, 정산

## 코딩 컨벤션
- 네이밍 규칙:      # 예: camelCase, 클래스는 PascalCase
- 패키지 구조:      # 예: controller / service / repository / entity / dto
- 예외 처리:        # 예: CustomException 클래스 사용, GlobalExceptionHandler로 처리
- 주석 스타일:      # 예: Javadoc 형식, 복잡한 비즈니스 로직에만 주석

## 자주 쓰는 패턴
예시:
- Service 레이어에서 트랜잭션 처리
- DTO ↔ Entity 변환은 MapStruct 사용
- API 응답은 CommonResponse<T> 래퍼 클래스로 통일

## AI에게 원하는 답변 방식
예시:
- 코드는 항상 전체 클래스 단위로 보여줘
- 의존성 주입은 생성자 주입 방식으로 해줘
- 테스트 코드도 같이 작성해줘 (JUnit5, Mockito)

## 자주 겪는 문제 유형
예시:
- JPA N+1 문제
- 트랜잭션 롤백 이슈
- 복잡한 동적 쿼리 (QueryDSL)
